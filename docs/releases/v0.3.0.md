# v0.3.0 Scope and Release Gate

`v0.3.0` is a stabilization release for the current DSL and release flow.

The goal is to make the existing behavior easier to trust, easier to learn, and easier to ship without growing the syntax surface in uncontrolled ways.

## Release Goals

- lock down the current DSL shape with explicit documentation
- improve parser diagnostics for common authoring mistakes
- protect representative README behavior with tests
- reduce release/test mismatch with cross-platform CI smoke coverage
- clarify install, release, and checksum guidance for end users

## Non-Goals

- no broad new DSL feature wave
- no major grammar redesign
- no platform/package-manager expansion beyond the release workflow
- no roadmap work for later versions in this document

## In Scope

- DSL specification and cheat sheet
- README onboarding and install/release clarity
- parser error-message improvements
- README-driven regression tests
- Linux full test coverage plus macOS/Windows smoke coverage
- release checklist and release gate clarity

## Out of Scope

- new receiver families or major syntax additions
- large refactors unrelated to stabilization
- packaging outside the current GitHub Releases artifacts
- speculative future-version planning

## Exit Criteria / Release Gate

Ship `v0.3.0` when all of the following are true:

- [ ] `README.md`, `docs/spec.md`, and `docs/cheatsheet.md` describe the current behavior clearly enough for first-time users
- [ ] parser diagnostics for common DSL mistakes are covered by tests
- [ ] representative README examples are covered by regression tests
- [ ] CI runs `cargo test --locked` on Linux and smoke checks on the non-Linux release platforms
- [ ] release workflow still builds and packages the supported artifacts with `.sha256` files
- [ ] install guidance matches the actual distribution path: GitHub Releases, `cargo install --locked`, and source build
- [ ] no known must-fix regression remains in the current DSL, docs, CI, or release flow
